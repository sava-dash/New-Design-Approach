    04 Process shift

    What is changing

    We design the product logic before we design the interface.

    The new process integrates agent exploration into the earliest stage of design, keeps knowledge in GitHub and Vercel for the team and client, and uses Figma as the place where validated UX flows turn into crafted UI.

    Research, exploration, craft, and delivery inside one workflow FuselabCreative

    Before Wireframes

    Design starts with project materials, not screens.

    Before any wireframing begins, the designer reviews briefs, requirements, stakeholder notes, references, and technical constraints to understand what the product must do.

    05 Project analysis before wireframing

    Shared Exploration Stack

    ClaudeCode or Codex become the first exploration partner.

    Exploration is no longer isolated in notes. Findings, prompts, structures, and early artifacts are stored in GitHub and shared through Vercel so colleagues and clients can follow progress.

    Approach summary The agent helps explore faster, but GitHub and Vercel make the work visible, reviewable, and reusable. Instead of keeping early thinking in private files, we create a living project space where the team can track decisions and the client can see momentum earlier.
    06 Shared workspace for design exploration

    Working Loop

    We move from CLI Agent to Figma and back again.

    The exploration phase is iterative: the designer tests ideas in ClaudeCode or Codex, translates the best structure into Figma, and returns to the agent to refine flows, states, and gaps until the UX direction is solid.

    07 Iterative loop between CLI exploration and Figma

    Phase 01

    UX flow proves the product structure.

    By the time the UX flow is approved, the navigation logic, key task paths, branches, and major states have already been explored and pressure-tested.

    01

    Material review

    Briefs, references, constraints, and existing product knowledge are collected first.

    02

    Agent exploration

    ClaudeCode or Codex helps map routes, states, IA options, and edge cases quickly.

    03

    Figma flow draft

    The strongest path is translated into UX flow screens and iterated until it is clear.

    08 Validated UX flow before visual styling

    Phase 02

    Manual UI builds on approved UX.

    Once the flow is ready, visual design does not start from zero. It builds on a stable UX foundation, which makes system thinking more intentional and screen production more consistent.

    01

    Foundations

    Typography, color, spacing, states, and rules are defined as a reusable system.

    02

    Components

    Manual UI craft turns the flow into purposeful screens and reusable interface parts.

    03

    Apply to UX

    The design system is then applied to the previously created UX flow for a coherent product experience.

    09 Manual UI craft plus design system application

    Phase 03

    GitHub handoff makes delivery build-ready.

    Delivery is not just a Figma file. Because GitHub and Vercel are connected during the process, the final handoff includes working structure, shared history, and a strong implementation head start.

    01

    GitHub repository

    Exploration notes, decisions, and implementation-ready artifacts stay organized in one place.

    02

    Vercel previews

    Colleagues and clients can review progress through live previews instead of waiting for static exports.

    03

    Strong handoff

    The client gets a near-ready product foundation that supports faster development and clearer collaboration.

    10 GitHub and Vercel powered client handoff

    Final Outcome

    A process that gives the team clarity and gives the client momentum.

    The result is a more informed start, a stronger UX foundation, a more intentional UI system, and a handoff that is much closer to a buildable product.
